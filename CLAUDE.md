# CLAUDE.md

This file provides guidance to Claude Code when working with code in this repository.

## 프로젝트 개요

HipLab — 힙합 장르 탐색 + 트렌드 검색 + 비트/가사 생성 서비스.
힙합을 배우면서 직접 만들어보는 "장르 실험실" 콘셉트.

## 주요 기능 (계획)

1. **장르 탐색기**: 붐뱁, 트랩, 드릴, 레이지, G-펑크, 퐁크, 로파이 등 설명 + 대표곡
2. **트렌드 탐색기**: 국내/해외 힙합 트렌드, 아티스트, 신곡 검색
3. **프롬프트형 작곡기**: 장르 스타일 기반 비트 구조 + 가사 생성
4. **가사 실험실**: 템플릿 기반 가사 생성 (허세형, 내면고백형, battle형 등)

## 자동 커밋/푸시 규칙

**모든 논리적 작업 단위가 완료되면 즉시 커밋하고 푸시한다.**

논리적 작업 단위 예시:
- 새 파일/컴포넌트 생성
- 기능 하나 구현 완료
- 버그 수정
- 설정 파일 변경
- 리팩토링

커밋 규칙:
1. 작업 단위 완료 시 `git add` → `git commit` → `git push` 자동 수행
2. 커밋 메시지는 한국어로 작성 (예: "feat: 장르 탐색기 페이지 추가")
3. 커밋 메시지 prefix: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`, `style:`, `test:`
4. 사용자 확인 없이 바로 커밋/푸시 (단, 위험한 작업 제외)
5. `.env`, 비밀키 등 민감한 파일은 절대 커밋하지 않음

## 코드 원칙

- **Simplicity First**: 최소한의 코드로 목표 달성
- **No AI Slop**: AI가 만든 촌스러운 패턴 금지 (불필요한 주석, 과도한 추상화 등)
- **Mobile First**: 모바일 반응형 우선 설계
